✅ 2. Levantar Requisitos Funcionais (RF) e Não Funcionais (RNF)
Requisitos Funcionais (RF)

São as funcionalidades que o sistema deve realizar.

1. Cadastro e Autenticação de Usuários
RF01: O sistema deve permitir o cadastro de usuários com e-mail, senha, nome de usuário e dados opcionais (bio, localização, ideologia).
RF02: O sistema deve autenticar usuários por login com senha ou redes sociais (OAuth).

2. Feed de Notícias e Conteúdo
RF03: O sistema deve exibir um feed com notícias de fontes verificadas (via RSS, APIs ou curadoria).
RF04: O sistema deve permitir ao usuário personalizar o feed com base em interesses políticos, temas ou fontes.

3. Publicações e Interações
RF05: O usuário deve poder publicar textos, imagens, links ou vídeos com comentários.
RF06: O sistema deve permitir curtir, comentar, compartilhar e reagir às publicações.

4. Sistema de Debates Políticos
RF07: O sistema deve permitir a criação de tópicos de debate (ex: "Voto obrigatório", "Privatizações").
RF08: O sistema deve permitir debates estruturados com sistemas de a favor/contra, votação e comentários.
RF09: O sistema deve destacar os comentários mais votados ou com melhor engajamento.

5. Grupos e Comunidades
RF10: O usuário deve poder criar ou entrar em grupos baseados em ideologias, movimentos ou temas.
RF11: Os grupos devem ter moderação, regras e tópicos de discussão próprios.

6. Sistema de Notificações
RF12: O sistema deve notificar o usuário sobre curtidas, respostas, novas postagens em tópicos seguidos e notícias de interesse.

7. Moderação e Reporte
RF13: O sistema deve permitir denunciar conteúdos ofensivos ou falsos.
RF14: Deve existir um painel de moderação para revisar denúncias e aplicar sanções.

8. Sistema de Reputação ou Pontuação
RF15: O sistema pode atribuir pontuação aos usuários com base em participação, qualidade de argumentos ou votos recebidos.

9. Busca Avançada
RF16: O sistema deve permitir buscar postagens, debates, usuários e grupos com filtros (ideologia, data, relevância).

10. Acessibilidade
RF17: O sistema deve estar disponível para dispositivos desktop com interface responsiva.

Requisitos Não Funcionais (RNF)

São as qualidades que o sistema deve ter (como segurança, desempenho etc.).
1. Desempenho
RNF01: O tempo de carregamento de páginas e feeds deve ser inferior a 2 segundos.
RNF02: A plataforma deve suportar picos de acesso simultâneo (escalabilidade horizontal).

2. Segurança
RNF03: Os dados do usuário devem ser armazenados com criptografia.
RNF04: O sistema deve ter proteção contra ataques comuns (XSS, SQL Injection, CSRF).
RNF05: Deve haver autenticação de dois fatores (opcional) e mecanismos de recuperação de senha.

3. Disponibilidade
RNF06: A plataforma deve estar disponível 99,9% do tempo (alta disponibilidade).

4. Usabilidade
RNF07: A interface deve ser intuitiva, clara e adaptável a diferentes perfis de usuário.
RNF08: Deve haver suporte a múltiplos idiomas (inicialmente PT-BR e EN).

5. Manutenibilidade
RNF09: O sistema deve ser modular e com código organizado para facilitar atualizações.

6. Privacidade e LGPD
RNF10: O sistema deve estar em conformidade com a LGPD (ou GDPR), oferecendo ao usuário controle sobre seus dados e opção de exclusão.

7. Portabilidade
RNF11: O app deve funcionar em Android, iOS e navegadores modernos.

8. Interoperabilidadee
RNF12: Deve ser possível integrar com APIs externas de notícias (ex: GNews, NYTimes) e redes sociais (Twitter/X, Mastodon, etc.).
